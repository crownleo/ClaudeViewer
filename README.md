# 🗂️ Claude 数据查看器 v4.0

> 本地运行 · 零安装 · 数据不离开你的设备

一个单文件 HTML 工具，用于查看和分析从 Claude.ai 导出的个人数据。双击即用，无需服务器、无需网络、无需账号。

---

## 🆕 v4.0 核心特性

### 📋 搜索结果侧边栏
对话内搜索后，点击右上角 📋 按钮展开侧边栏，一次看到所有命中位置（发送者、片段、消息序号），点击任意结果直接定位。

### 📅 活动热力图
统计页内类似 GitHub 贡献图的按天活动可视化。颜色深浅表示当天对话量，悬停查看具体数字，点击某天筛选出当天对话，列表顶部提供「✕ 清除」恢复全部。

### ⤒⤓ 跳转顶部/底部
长对话阅读时，消息区右下角悬浮按钮，平滑滚动到顶部或最新消息，滚动时智能显示/隐藏。

### 📑 代码块一键复制
所有 Markdown 代码块右上角增加复制按钮，hover 时显示，点击后变「已复制 ✓」。

### 💾 本地持久化缓存（IndexedDB）
导入数据后可选保存到浏览器本地，下次打开无需重新导入。隐私优先设计：
- 首次导入弹窗让你选择「保存到本地」或「仅本次使用」
- 可勾选「记住我的选择」跳过后续询问
- 侧边栏底部常驻缓存状态，「清除」按钮永远可见
- 数据仅存于本机浏览器，不上传任何地方

### 🗂 项目与记忆职责分离
- **📁 项目** Tab：显示项目信息 + System Prompt + 文档 + 该项目的项目记忆
- **🧠 记忆** Tab：专门展示手动导入的全局记忆（`.md`），不再与项目记忆重复

## 🚀 快速开始

### 第一步：获取 Claude 导出数据
1. 打开 [claude.ai](https://claude.ai) 并登录
2. 头像 → **Settings** → **Privacy** → **Export data**
3. 点击 **Export**，等待邮件通知（通常几分钟内）
4. 下载邮件中的 `.zip` 压缩包

### 第二步：打开查看器
双击 `claude_viewer_v4.html`，在浏览器中打开。

> **推荐浏览器**：Chrome / Edge  
> Safari 可正常查看对话，但 PDF 导出功能受限

### 第三步：导入数据
将 `.zip` 文件**直接拖入**页面，自动解析全部内容。导入成功后可选择是否保存到本地。

## 🔒 隐私说明

- **完全本地运行**：所有数据仅在浏览器内处理，不经过任何服务器
- **网络请求**：仅加载两个 CDN 库（marked.js、JSZip），不传输任何个人数据
- **默认不持久化**：除非你主动选择「保存到本地」
- **IndexedDB 缓存**：选择保存后数据存于此设备浏览器（`file://` origin 绑定），仅本机可读，可随时清除
- **localStorage 存储**：收藏列表、标签、深色模式、缓存偏好（不含对话内容）

## 📦 技术栈

- 纯原生 HTML / CSS / JavaScript，无框架，无构建工具
- [marked.js 9.1.6](https://marked.js.org/) — Markdown 渲染
- [JSZip 3.10.1](https://stuk.github.io/jszip/) — ZIP 解析与生成
- 虚拟滚动：自实现（绝对定位 + requestAnimationFrame）
- 统计图表 / 热力图：内联 SVG + DOM，无第三方图表库
- 本地持久化：IndexedDB
- 深色模式：CSS 变量 + `data-theme` 切换

## 📁 导出包文件说明

| 文件 | 内容 |
|---|---|
| `conversations.json` | 全部对话（消息、时间戳、Thinking、附件） |
| `users.json` | 账户基本信息 |
| `memories.json` | 项目记忆数据（在「项目」Tab 对应项目下查看） |
| `projects/{uuid}.json` | 项目元数据（名称、System Prompt、文档） |

> 全局记忆是 Claude 跨对话记住的用户信息，不在标准导出包内，需在 Claude 界面单独导出为 `.md` 后导入查看器的「记忆」Tab。
