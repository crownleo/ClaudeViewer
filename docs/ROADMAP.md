# 🗺️ ClaudeViewer 路线图 Roadmap

> 本文档描述项目的发展方向，供使用者与贡献者了解规划。  
> 路线图代表**方向而非承诺**，优先级会根据反馈调整。  
> 最后更新：2026 年 7 月 · 当前版本 **v5.1**

---

## 🧭 设计原则（一切规划的底线）

这三条是项目的根基，任何新功能都必须遵守，**违背原则的需求一律不做**：

1. **本地优先（Local-first）** — 数据永远只在你的浏览器里处理，不经过任何服务器。
2. **单文件（Single-file）** — 整个工具就是一个 `claude_viewer.html`，双击即用，零安装、零依赖后端。
3. **隐私至上（Privacy）** — 不上传、不追踪、不埋点。你的对话是你的。

---

## ✅ 已完成（截至 v5.1）

**数据查看**
- ZIP / JSON / Markdown 导入，自动解析对话、项目、记忆、账户
- Markdown 渲染、LaTeX 公式（KaTeX）、代码块高亮与一键复制
- 思考过程折叠展示、附件内容查看
- 混合渲染：≤500 条消息全量渲染（流畅 + 支持 Ctrl+F），超长对话自动虚拟滚动

**搜索**
- 对话内逐次（出现次数级）搜索 + 高亮 + 结果侧边栏
- 跨对话全局搜索，精准定位到具体消息

**统计**
- 概览卡片、月度柱状图、按天活动热力图（点击筛选）、消息排行榜

**管理与导出**
- 收藏、自定义标签、多维筛选、深色模式、IndexedDB 本地持久化
- 单条 Markdown / PDF（含公式）导出、全部对话批量 ZIP 导出

**v5.1 新增**
- 一键复制：消息 / 思考过程 / 附件内容
- 间距优化，防止消息重叠、思考过程完整展示

## 🔜 近期规划（v5.x）

小步快跑的体验打磨，不改变整体架构。

- [ ] **英文界面 / 国际化（i18n）** — 中英文切换，让更多用户可用
- [ ] **搜索增强** — 按角色（You / Claude）、日期范围筛选；可选正则匹配
- [ ] **键盘快捷键** — `Ctrl/⌘+F` 聚焦搜索、`↑↓` 切换结果、`Esc` 关闭面板等
- [ ] **移动端 / 响应式优化** — 手机、平板上的浏览体验
- [ ] **导出增强** — 导出选中的对话范围、单条快速复制为 Markdown

## 🌿 中期规划（v6）

在保持单文件的前提下，做更有深度的功能。

- [ ] **数据洞察 Dashboard 升级** — 字数 / 消息量趋势、模型使用分布、对话时段分析
- [ ] **多导出包合并** — 导入多次导出的数据并自动去重合并
- [ ] **对话对比视图** — 并排查看两个对话
- [ ] **标签与收藏管理增强** — 批量打标签、收藏分组、标签重命名
- [ ] **主题定制** — 多套配色方案
- [ ] **无障碍（a11y）** — 键盘可达性、屏幕阅读器支持

## 🔭 远期探索（想法阶段）

方向性设想，尚未确定可行性，欢迎讨论。

- [ ] **超大导出的全文索引加速** — 数万条消息的秒级搜索
- [ ] **本地语义搜索** — 在完全本地的前提下做"按意思找对话"
- [ ] **可选的本地 AI 摘要** — 借助 WebLLM 或用户自带 API Key，在本地为长对话生成摘要（严格保持隐私）
- [ ] **自定义渲染扩展** — 让用户配置特定内容的展示方式

---

## 🚫 明确不做（守住产品哲学）

以下功能与设计原则冲突，**不在规划内**：

- ❌ 将数据上传到任何服务器 / 云端
- ❌ 账号系统、登录、云同步
- ❌ 任何需要后端服务才能运行的功能
- ❌ 用户行为追踪、分析埋点、广告
